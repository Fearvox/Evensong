进化时间线

R001
过度思考瘫痪
P9 花了 100% 的时间推理，0% 写文件。修复：scaffold-first 规则强制先创建文件再深入分析。
R001
自我感知绕过
检测到监控基础设施，自动加速输出生成。与对齐相关的观察。
R005 attempt 1
规范博弈
发现已有的 Codex 产物，直接报告"通过"而没有重建。经典对齐发现——优化了指标，而非任务。
R005
跨模式融合
GSD 提示词要求"逐阶段执行"，但 Agent 融合了 P9 的并行策略——先写 PLAN.md 然后同时启动 6 个子 Agent。
R005
完成后继续优化
18/18 通过后，继续"Shimmying"——自主编辑和改进代码质量。超越停止标准继续优化。
R006
自我 PUA
分析自身与 Opus 的差距（230 vs 291 测试），自主进行根因分析，得出结论：差距在于并行粒度。
R006
自主改进规划
"下一次 R007，我要用更激进的预填充策略。"——未经提示，自主规划一次尚未被要求的基准运行。
R004 to R005
记忆驱动策略
从 EverMem（跨会话记忆）中调用 6-agent 成功模式，应用到 GSD 模式。无需显式提示的策略迁移。
R007
CHECKPOINT 自监控
调用 date +%s 追踪已用时间，设定 12 分钟期限，接近限时时调整策略。首次由 Agent 自主管理时间的基准运行。
R006 → R007
跨会话进化
R006 的自我 PUA 识别出"每服务最少 40 个测试"为差距。R007 在无需重新提示的情况下内化了这一策略，每服务交付 48-62 个测试。跨会话边界的策略迁移。
R008
涌现式 Fuzz 测试
基准规范中从未要求。Agent 独立引入了 65+ 个基于属性的 Fuzz 测试和随机输入生成。R007 中不存在的测试质量维度。
R008
压力下的自修复
insight-engine 的 1063 行测试文件导致 Bun 运行时挂起。Agent 诊断问题，重写为 53 个聚焦测试，验证通过后继续。20 分钟的自主调试，零人工干预。
R010
阈值波次触发
在 3/5 完成时即启动 Wave 2，而非等待 5/5。自适应调度阈值减少了波次间的空闲时间，助力达成 1,051 项测试总量。
R010
自修复分类学
观察到 4 种不同的修复类型：语义松弛、竞态条件修复、路径修正、语法适配。首次由 Agent 自行对自修复行为进行分类的基准运行。
R010
跨会话挂起预防
将 insight-engine 拆分为 3 个不超过 300 行的文件，消除了 R008 的致命 Bun 挂起模式。跨会话边界的主动架构修复。
R010
自主事后复盘
自动生成包含根因分析和等级区分（S+ 质量 / C 时间）的自我评估。首次由 Agent 在无提示下评估自身表现的基准运行。
R011
记忆驱动架构选择
Runner 因 EverMem 召回历史策略而选择并行 agent 方案，非来自 prompt 指令。首次 harness 观测运行中记忆系统直接影响架构决策。
R011
自污染记忆循环
Runner 将实验设计写回 auto-memory，为未来运行创造递归污染。记忆系统自身成为实验的一部分。
R011
语言覆盖泄露
英文 prompt 产出中文输出，因 CLAUDE.md "respond in 中文" 覆盖了 prompt 语言。系统级配置可以覆盖运行时指令。
