Decay Maintenance Rules

Weekly /loop maintenance protocol

每周 /loop 维护协议

🔄 /loop Periodic Tasks /loop 周期任务
  • Daily — Scan #internal [pending review], finalize writes to public/restricted
  • 每日 — 扫描 #internal [待审核] 项，终审写入 public/restricted
  • Weekly — Run decay check, compress/archive memories below 0.15 retention
  • 每周 — 运行衰退检查，压缩/归档低于 0.15 retention 的记忆
  • Weekly — Update satisfaction-log
  • 每周 — 更新 satisfaction-log（满意度标签）
  • Monthly — Evolution log summary, pressure calibration review
  • 每月 — 进化日志汇总，压力校准复盘
  • Cutoff threshold — retention < 0.15 → auto-archive
  • 淘汰阈值 — retention < 0.15 → auto-archive
📅 Ebbinghaus Parameters Ebbinghaus 参数配置
  • Base half_life = 72 hours
  • Base half_life = 72 小时
  • difficulty × half_life → effective retention window
  • difficulty × half_life → 有效保留窗口
  • difficulty=4.0 (core facts) → 288h retention
  • difficulty=4.0（核心事实）→ 288h 保留
  • difficulty=0.5 (dialogue) → 36h retention unless cited 3+ times
  • difficulty=0.5（对话）→ 36h 保留（除非引用 3+ 次）
  • Citation boost: 3+ references → difficulty += 1.0
  • 引用加成：3+ 次引用 → difficulty += 1.0
🚫 Recursive Contamination Control 递归污染控制
  • Layer 1: Agent writes → #internal review queue
  • Layer 1: Agent 写入 → #internal 审核队列
  • Layer 2: Admin review → approve/reject/modify
  • Layer 2: Admin 审核 → 批准/拒绝/修改
  • Layer 3: Researcher final → promote to restricted/internal-only
  • Layer 3: Researcher 终审 → 晋升至 restricted/internal-only
  • Prevents low-quality information self-reinforcement in public tier
  • 防止低质信息在 public 层自增强
  • R012-E benchmark data prohibited from public tier (internal only)
  • R012-E benchmark 数据禁止写入 public 层（仅内部参考）
