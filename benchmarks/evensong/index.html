Tool Reference

15 工具详解

点击展开各工具使用规范与参数

⏱️
convert_timestamp
时间转换 / 当前时间基准

获取当前时间基准 / 时间格式互转 / 时间戳转可读时间

  • time_now_str — 当前时间字符串
  • time_yesterday_str — 昨天时间（当前-24h）
  • time_week_str — 一周前时间（当前-7d）
  • parsed_time_unix — UTC 时间戳（API 直用）
Asia/Shanghai ISO 8601 UTC+8
📆
schedule_meeting
创建会议

必须获取：subject（主题）、start_time、end_time

  • 未指定结束时间 → 默认 1 小时
  • 周期性会议：需 meeting_type=1 + recurring_type + until_count
  • 不支持邀请人，创建成功后不返回邀请人
强制 subject 周期性
✏️
update_meeting
修改会议

修改前必须二次确认，展示会议信息，用户确认后再执行

  • 展示原会议信息（subject、start_time、end_time）
  • 等待用户明确确认
  • 返回 X-Tc-Trace 追踪字段
二次确认 敏感操作
🗑️
cancel_meeting
取消会议

取消前必须二次确认，展示会议信息

  • 取消子会议 → 传 sub_meeting_id
  • 取消整场 → 传 meeting_type=1
  • 返回 rpcUuid 用于问题排查
二次确认 不可逆
🔎
get_meeting
查询会议详情 (meeting_id)

通过 meeting_id 查询会议详情

  • 返回主持人和参会者昵称（默认不返回用户 ID）
  • 周期性会议含 current_sub_meeting_id
  • 需传入 _client_info 对象
meeting_id 周期性
🔢
get_meeting_by_code
会议号 → meeting_id

将 9 位会议号（meeting_code）转换为 meeting_id

  • 用户给会议号时，先调用此工具
  • 获取 meeting_id 后再调用目标工具
  • 作为录制/转写查询的前置步骤
前置步骤 必调用
👥
get_meeting_participants
获取参会成员明细

查看实际参会人员、参会明细

  • 周期性会议必须传 sub_meeting_id
  • 根据 has_remaining 判断是否分页
  • 下一页用返回的 next_pos
分页查询 成员明细
🎬
get_records_list
查询录制列表

查询会议录制文件列表

  • 传 meeting_id 或 meeting_code → 时间可不传
  • 未传会议 ID → start_time 和 end_time 必须同时传
  • 返回 meeting_record_id
meeting_id 录制
📥
get_record_addresses
获取录制下载地址

获取录制文件下载链接

  • 用户提供会议号时执行三步：
    ① get_meeting_by_code
    ② get_records_list
    ③ get_record_addresses
  • 返回多个 resolution 备选地址
三步查询 下载地址
📝
get_transcripts_details
查询转写详情

获取会议转写全文

  • 分页：pid（起始段落 ID）+ limit
  • 用户给会议号时三步：
    ① get_meeting_by_code
    ② get_records_list → record_file_id
    ③ get_transcripts_details
转写全文 分页
🧠
get_smart_minutes
获取智能纪要

获取 AI 生成的智能会议纪要

  • 推荐信息获取优先级：
    ① get_smart_minutes（优先）
    ② get_transcripts_details
    ③ get_record_addresses
  • lang 支持：default / zh / en / ja
AI 纪要 多语言
🔄
check_skill_version
版本更新检查

检查是否有新版本可用

  • 服务端强制拦截版本过旧请求
  • 根据返回下载链接安装最新版本
  • 禁止修改请求头 X-Skill-Version 绕过检查
  • 更新后必须重新开始新对话
强制更新 版本管理

Critical Rules

核心规范

使用前必须阅读，冲突时以本文为准

⏰
时间处理规范
Asia/Shanghai · ISO 8601
  • 默认时区：Asia/Shanghai (UTC+8)
  • 用户说"今天/明天/下周一" → 必须先调用 convert_timestamp
  • 禁止依赖模型猜测当前时间
  • 跨时区：传 timezone 参数，parsed_time_unix 已是 UTC，禁止二次转换
⚠️
敏感操作规范
二次确认 · X-Tc-Trace
  • 修改/取消会议前 → 展示会议信息 + 二次确认
  • 所有工具返回的 X-Tc-Trace / rpcUuid → 必须展示给用户
  • 调用所有工具必须附带 _client_info 对象（os / agent / model）
  • 会议号（9位）→ 先 get_meeting_by_code 转 meeting_id
