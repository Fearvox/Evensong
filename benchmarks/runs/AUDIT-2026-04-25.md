# Benchmark Runs Public Artifact Audit

Date: 2026-04-25

Scope: `benchmarks/runs/` plus benchmark registry references that point back into run artifacts.

## Findings

- No zero-byte files were present.
- 59 `pre-snapshot.json` / `post-snapshot.json` files contained only `[]`; these were placeholder artifacts, not evidence.
- One tracked desktop screenshot exposed local workstation context and command state.
- Run transcripts and dense-retrieval reports exposed local absolute paths, remote root paths, and a private embedding endpoint.
- JSON and JSONL artifacts were structurally valid before cleanup.

## Remediation

- Redacted local workspace paths to `<REPO_ROOT>`.
- Redacted local-only paths outside the repo to `<LOCAL_PATH_REDACTED>`.
- Redacted remote repo/vault roots to `<REMOTE_REPO>` and `<REMOTE_RESEARCH_VAULT>`.
- Redacted private embedding service URLs to `<NON_LOCAL_PROVIDER_ENDPOINT>`.
- Removed 51 tracked empty snapshot placeholder files.
- Removed 8 local untracked empty snapshot placeholders so they cannot be accidentally added later.
- Removed the tracked desktop screenshot from `R028-gemini-clean`.
- Removed local `.DS_Store` residue.
- Redacted benchmark registry transcript paths that mirrored the same local absolute path.

## Verification

- `benchmarks/runs` now contains 620 files, including this audit report.
- JSON files: 256.
- JSONL files: 129.
- JSONL rows parsed: 5705.
- JSON/JSONL parse errors: 0.
- Small files `<=10` bytes: 0.
- Sensitive scan for local paths, remote root paths, private IP endpoints, bearer tokens, and `sk-*` keys: 0 matches in `benchmarks/runs` and benchmark registry references.
- Repository-wide benchmark scan has no remaining local path/private endpoint hits.

## Residual Notes

- Synthetic test passwords and toy secrets inside generated benchmark code remain as benchmark fixture content, not real credentials.
- Raw transcripts remain by design because they are the evidence trail; only environment-specific routing details were redacted.
- Runs with zero tests or invalid status are retained as historical benchmark outcomes rather than deleted.
