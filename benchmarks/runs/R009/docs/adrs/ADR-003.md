# ADR-003: JWT Simulation for Auth (No Real Crypto)

**Status:** Accepted
**Date:** 2026-04-09

## Context

The auth-gateway service must issue tokens and all other services must validate them on incoming requests. Using a real JWT library (e.g. `jose`, `jsonwebtoken`) requires either a live key pair or a shared symmetric secret, introduces non-trivial CPU cost for signing/verification across hundreds of test calls, and can fail with subtle timing or environment issues. The benchmark goal is to test authorization logic (role checks, scope enforcement, token expiry handling) — not cryptographic correctness.

## Decision

The auth-gateway issues tokens as `base64(JSON.stringify(payload))` with no signature in benchmark mode. The middleware in each service's request pipeline calls `parseSimulatedToken(header)` from `@evensong/auth/mock`, which base64-decodes the header value and returns the payload object. Role and scope checks run against this payload normally. A `BENCHMARK_AUTH=true` environment variable gates this behavior; in any other environment the real `jose` implementation is used.

## Consequences

Token issuance and validation cost drops to microseconds, removing crypto from the benchmark's performance profile. Authorization logic (the actual business concern) is tested faithfully. The explicit env gate ensures the mock path cannot be activated in staging or production by accident. The tradeoff is that token forgery resistance, algorithm selection, and key rotation are not validated by any test in this suite.
