Core offer

Turn raw agent capability into observable work by changing the surrounding harness, not just the model.

Positioning

The simple version

Evensong is not a claim that agents are conscious. It is a claim that persistent context, pressure, and evaluation design materially change agent behavior.

Pressure

Agents ship more when the harness makes work visible

Time checks, completion criteria, and self-monitoring changed how agents allocated effort across services and tests.

Memory

Strategy recall changes future runs

When prior patterns are available, agents reuse or adapt them. The result is a different plan before the first file is written.

Proof

False output is measured, not hidden

Fabricated runs remain in the public story because they explain why clean-room protocols and evidence trails matter.

Public copy

Use this framing

The model is only one part of the system. Evensong studies the work loop around it: prompts, memory, pressure, tools, verification, and the way a run is allowed to stop.

Boundary: Public pages can promote the operational lesson, but gated R012-E details remain internal self-evolution reference. Do not turn internal vault material into public claims.
