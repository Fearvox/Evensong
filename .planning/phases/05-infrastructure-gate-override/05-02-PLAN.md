    **Test strategy:** Create a minimal HTTP server in-process using Bun.serve() that implements the MCP SSE protocol. The server:
    1. Accepts GET on `/sse` -- responds with `Content-Type: text/event-stream`, sends an `endpoint` event with a POST URL
    2. Accepts POST on `/message` -- receives JSON-RPC requests, responds with JSON-RPC results

    This avoids external dependencies and tests SSE transport under Bun natively.

    **Step 1: Create in-process SSE server**

    ```typescript
    import { describe, test, expect, beforeAll, afterAll } from 'bun:test'
    import { Client } from '@modelcontextprotocol/sdk/client/index.js'
    import { SSEClientTransport } from '@modelcontextprotocol/sdk/client/sse.js'
    import { ListToolsResultSchema, CallToolResultSchema } from '@modelcontextprotocol/sdk/types.js'

    let server: ReturnType<typeof Bun.serve>
    let serverUrl: string

    beforeAll(() => {
      server = Bun.serve({
        port: 0, // random available port
        fetch(req) {
          const url = new URL(req.url)

          if (url.pathname === '/sse' && req.method === 'GET') {
            // SSE endpoint -- send endpoint event, then keep stream open
            const stream = new ReadableStream({
              start(controller) {
                const endpoint = `http://localhost:${server.port}/message`
                controller.enqueue(`event: endpoint\ndata: ${endpoint}\n\n`)
              }
            })
            return new Response(stream, {
              headers: {
                'Content-Type': 'text/event-stream',
                'Cache-Control': 'no-cache',
                'Connection': 'keep-alive',
              }
            })
          }

          if (url.pathname === '/message' && req.method === 'POST') {
            return (async () => {
              const body = await req.json()
              // Handle JSON-RPC requests
              if (body.method === 'initialize') {
                return Response.json({
                  jsonrpc: '2.0', id: body.id,
                  result: {
                    protocolVersion: '2024-11-05',
                    capabilities: { tools: {} },
                    serverInfo: { name: 'sse-test-server', version: '1.0.0' }
                  }
                })
              }
              if (body.method === 'tools/list') {
                return Response.json({
                  jsonrpc: '2.0', id: body.id,
                  result: {
                    tools: [{
                      name: 'sse_test_tool',
                      description: 'SSE test tool',
                      inputSchema: { type: 'object', properties: { input: { type: 'string' } } }
                    }]
                  }
                })
              }
              if (body.method === 'tools/call') {
                const args = body.params?.arguments || {}
                return Response.json({
                  jsonrpc: '2.0', id: body.id,
                  result: {
                    content: [{ type: 'text', text: 'sse-echo: ' + (args.input || 'none') }]
                  }
                })
              }
              return Response.json({ jsonrpc: '2.0', id: body.id, error: { code: -32601, message: 'Method not found' } })
            })()
          }

          return new Response('Not found', { status: 404 })
        }
      })
      serverUrl = `http://localhost:${server.port}/sse`
    })

    afterAll(() => { server?.stop() })
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP server process -> CLI | MCP server is a child process; its tool definitions are untrusted input |
| MCP config file -> transport | settings.json MCP config specifies command to spawn; trust boundary is the config file |
| MCP SSE server -> CLI | SSE server is a remote HTTP endpoint; responses are untrusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | MCP tool definitions | accept | Tool definitions from MCP servers are displayed to user before execution; existing permission system gates tool calls. MCP is user-configured. |
| T-05-05 | Tampering | MCP server command | accept | MCP server command comes from settings.json (user-writable config). This is by design -- user chooses which MCP servers to trust. Existing Zod schema validation in types.ts validates config shape. |
| T-05-06 | Denial of Service | MCP server hang | mitigate | Connection timeout already set in client.ts (configurable). Tests verify that non-existent commands fail fast, not hang. |
| T-05-10 | Spoofing | SSE transport endpoint | mitigate | SSE endpoint URL comes from user config (settings.json). Tests verify connection failure is catchable. In production, TLS should be used for remote MCP servers. |
</threat_model>
