<objective>
Make history writes abort-safe: ensure appendEntryToFile never produces partial JSON lines, and add a finalizeHistoryOnAbort helper that flushes accumulated state as a complete entry before the process exits.

Purpose: When a user aborts mid-stream (Ctrl+C, ESC), the history file must remain valid JSONL. Currently appendFileSync with O_APPEND is atomic for writes under PIPE_BUF (4096 bytes on POSIX), which covers most entries. The gap is: if the process is killed mid-stream, we need to ensure the last entry written is complete. This plan adds a finalize function and tests proving abort safety.

Output: Modified sessionStorage.ts with exported finalizeHistoryOnAbort, plus test file validating atomic append behavior.
</objective>

```typescript
// NOT exported -- internal function
function appendEntryToFile(
  fullPath: string,
  entry: Record<string, unknown>,
): void {
  const fs = getFsImplementation()
  const line = jsonStringify(entry) + '\n'
  try {
    fs.appendFileSync(fullPath, line, { mode: 0o600 })
  } catch {
    fs.mkdirSync(dirname(fullPath), { mode: 0o700 })
    fs.appendFileSync(fullPath, line, { mode: 0o600 })
  }
}
```

Key insight from research: appendFileSync with O_APPEND IS atomic for writes under PIPE_BUF (4096 bytes). The risk is process termination during a multi-step write sequence where several appendEntryToFile calls happen in a row (e.g., writing message + tool results). The fix is a finalizeHistoryOnAbort that ensures the last logical entry is flushed completely.
