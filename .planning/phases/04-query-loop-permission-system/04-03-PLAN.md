**必须实现的测试用例（QUERY-03）：**

首先：**尝试 loadTranscriptFile() 方式。** 如果 loadTranscriptFile() 需要 bootstrap 状态而崩溃，fallback 到仅测试 buildConversationChain()（使用手动构造的 Map）。

**方案 A：loadTranscriptFile() 直接测试（首选）**

1. **写入单条 user 消息并读回**：
   - 用 `writeFileSync` 写一行 JSONL：`{ type: 'user', uuid: 'u1', parentUuid: null, ... }`
   - 调用 `loadTranscriptFile(filePath)`
   - 断言 `result.messages.size === 1`
   - 断言 `result.messages.get('u1')?.type === 'user'`

2. **两条消息链（user → assistant）**：
   - 写两行 JSONL（user + assistant，assistant.parentUuid = user.uuid）
   - 调用 `loadTranscriptFile(filePath)`
   - 断言 `result.messages.size === 2`
   - 断言 `result.leafUuids.has('uuid-assistant')` — assistant 是 leaf
   - 取 leaf UUID，调用 `buildConversationChain(result.messages, leafMsg)`
   - 断言链长度 === 2，顺序为 [user, assistant]
   - 断言 chain[0].type === 'user'，chain[1].type === 'assistant'

3. **isTranscriptMessage 过滤**：
   - 用 'abort' 类型条目（非 TranscriptMessage）
   - 断言 `isTranscriptMessage({ type: 'abort', ... } as any)` 返回 false
   - 断言 `isTranscriptMessage({ type: 'user', ... } as any)` 返回 true

**方案 B：buildConversationChain() 内存测试（fallback，若 loadTranscriptFile 需要 bootstrap）**

如果 loadTranscriptFile 在无 bootstrap 环境下抛错，改用：
- 手动构造 `Map<string, TranscriptMessage>`
- 直接填入 user 消息和 assistant 消息
- 调用 `buildConversationChain(map, leafMsg)`
