<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test -> filesystem | Tests create temp dirs with fixture files; must clean up to prevent disk fill |
| Test -> ripgrep child process | GrepTool spawns rg as child process; test must ensure it terminates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Tampering | FileEditTool test temp files | accept | Tests use mkdtemp in OS temp dir with afterEach cleanup; no production files touched |
| T-02-05 | Denial of Service | GrepTool large result test | mitigate | Test creates exactly 20 small files (not thousands); head_limit=5 ensures bounded output. afterEach cleanup prevents disk fill. |
| T-02-06 | Information Disclosure | Path traversal in test inputs | accept | Tests use absolute paths within mkdtemp directory; no real project files are read/modified |
| T-02-07 | Tampering | Binary file creation in temp dir | accept | Binary fixtures contain non-sensitive random bytes; cleaned up in afterEach |
</threat_model>
