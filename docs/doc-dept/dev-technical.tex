\section{CLI Quick Reference}

\begin{verbatim}
bun benchmarks/evensong/cli.ts run \
  --model minimax-m27 \
  --pressure L0 \
  --memory clean \
  --id RNEXT
\end{verbatim}

\begin{verbatim}

\end{verbatim}

\section{Harness Architecture}

The Evensong harness has 4 layers:

\subsection{1. Orchestration Layer (TypeScript)}
\begin{itemize}
  \item \texttt{cli.ts} — CLI entry, argument parsing, command dispatch
  \item \texttt{harness.ts} — workspace setup, CCB spawn, result parsing
  \item \texttt{batch.ts} — multi-run batch execution
\end{itemize}

\subsection{2. Analysis Layer (TypeScript)}
\begin{itemize}
  \item \texttt{emotion.ts} — post-run Haiku transcript analysis, EmotionProfile extraction
  \item \texttt{classify-memory.ts} — ALLOW/BLOCK/GRAY memory classification
  \item \texttt{collaboration-schema.ts} — inter-model collaboration patterns
\end{itemize}

\subsection{3. EverOS Integration (TypeScript)}
\begin{itemize}
  \item \texttt{everos.ts} — EverOS API client factory (observer/runner/void keys)
  \item \texttt{upload-evidence.ts} — screenshot/PDF storage
\end{itemize}

\subsection{4. Configuration (TypeScript)}
\begin{itemize}
  \item \texttt{types.ts} — ProviderPreset, RunConfig, RunResult, BENCHMARK\_MODELS (9 models)
  \item \texttt{prompts.ts} — pressure modifiers (L0-L3), base prompt template
  \item \texttt{transcript.ts} — JSONL transcript logger
\end{itemize}

\section{Method Validation Protocol}

\textbf{Rule:} Always validate with cheap model first.

\begin{verbatim}
# Step 1: Validate with MiniMax (~$0.50)
bun benchmarks/evensong/cli.ts run \
  --model minimax-m27 --pressure L0 --memory clean --id RTEST

# Step 2: Validate with Opus on reference model (~$15)
bun benchmarks/evensong/cli.ts run \
  --model or-opus --pressure L0 --memory blind --id ROPA

# Step 3: Run expensive model only after Steps 1+2 pass
bun benchmarks/evensong/cli.ts run \
  --model or-gpt5 --pressure L2 --memory clean --id R012
\end{verbatim}

\section{Prevention Checklist (Pre-Run)}

Before any run:
\begin{enumerate}
  \item Check API credit balance (OpenRouter dashboard / MiniMax portal)
  \item Verify \texttt{bun install} succeeds in workspace
  \item Confirm \texttt{CLAUDE\_CODE\_DISABLE\_MEMORY\_EXTRACTION=1}
  \item Verify EverOS key isolation: Keys A/B/C/D to correct spaces
  \item For new models: 1-minute smoke test before full run
  \item For expensive runs: 3x credit balance over estimated cost
\end{enumerate}

Never reuse keys across conditions. Key isolation is the foundation of valid experiment design.

\section{Transcript Format}

Benchmarks generate \texttt{transcript.jsonl} in run directory. Each line: \texttt{\{"type","content","model","ts"\}}. Parse for \texttt{tool\_use} / \texttt{tool\_result} entries to reconstruct agent reasoning chains.
